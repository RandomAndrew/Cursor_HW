Домашнє завдання на тему: GIT та його команди

  • git config --global user.name- задати/змінити ім'я!.
  • git config --global user.email - задати/змінити email.
  • git init - ініціює пустий репозиторій в поточній репозиторії.
  • git status - показує стан файлів в робочій директорії та індексі.
  • git add - добавляє вміст робочої директорії в індекс для наступного комміта.
  • git commit - зберігає всі додані данні та зберігає в сліпок в внутрішній базі данних.
  • git log - використовується для перегляду історії коммітів.
  • git checkout - використовується для переключення гілок і вигрузки їх складу в робочу директорію.
  • git branch - перераховує гілки, створює нові, видаляє або змінює назву.
  • git push - використовується для встановлення зв'язку з віддаленим репозиторієм(тобто відправляє з локального).
  • git pull - працює як комбінація команд git fetch і git merge, git спочатку забирає зміни з вказаного видаленого репозиторія, а потім намагається злити їх з поточною гілкою.
  • git merge - використовується для злиття одної або декількох гілок в поточну.
  • git clone - процей створення ідентичної копії віддаленого репозиторія Git на локальну машину.

.gitignore - потрібен для приховання файлів або папок від системи контролю версій Git. Простіше сказати це текстовий файл, в кожному рядку якого міститься шаблон файла або каталога який необхідно проігнорувати.

ADVANCED

  • git revert-відміняє зміни занесені окремим коммітом.
  • git reset-складний універсальний інструмент для відміни змін.
  • git rebaseодин з способів об'єднати зміни, зроблені в одній гілці с іншою гілкою.
  • git cherry-pick-використовується для перенесення окремих коммітів з ондого місця репозиторія в інше.
  • git stash-використовується для того аби помістити поточні зміни, які ви виконали в файлах, в окреме сховище і повернути файли до початкового стану.
